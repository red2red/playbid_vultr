## 리뷰 체크리스트

### A. 정보 구조/레이아웃
- [ ] 사이드바 + 메인 콘텐츠 구조가 유지된다.
- [ ] Desktop은 좌 70% / 우 30%, Tablet은 좌 65% / 우 35%를 지킨다.
- [ ] Mobile은 단일 컬럼이며 하단 sticky CTA 바가 있다.
- [ ] Header에 breadcrumb, back 버튼, bookmark/share/print 액션이 포함된다.
- [ ] 타이틀 섹션에 카테고리 배지, 상태 배지, 공고명, 기관명, 메타데이터가 포함된다.

### B. 상세페이지 핵심 컴포넌트
- [ ] 좌측 컬럼에 `주요 정보`, `일정`, `상세 내용(탭)`, `첨부파일`, `유사 공고` 카드가 모두 존재한다.
- [ ] 우측 컬럼에 `빠른 액션`, `마감 정보`, `프리미엄 분석`, `참가업체 통계`, `유사 사정율` 카드가 모두 존재한다.
- [ ] 빠른 액션 CTA가 `모의입찰 시작`, `원문 보기`, `적격심사 계산`으로 구성된다.
- [ ] 프리미엄 카드에 가격/락 상태(비유료 사용자)가 표현된다.
- [ ] 유사 공고 영역이 스크롤 가능한 카드 리스트로 표현된다.

### C. Flutter 앱 일관성
- [ ] 공통 토큰(색상/폰트/라운드/간격)이 `Task.md T-012` 기준과 충돌하지 않는다.
- [ ] 핵심 데이터 필드 명칭/의미가 Flutter 계약(공고번호, 일정, 금액, 참가자격 등)과 일치한다.
- [ ] 북마크 UX(토글 즉시 피드백)가 Flutter와 동일한 기대 동작을 가진다.
- [ ] 로그인 필요 기능(모의입찰/유료 기능)의 게이팅 문구가 명확하다.
- [ ] 카테고리 표기에서 `product ↔ goods` 의미 불일치가 없다.

### D. 상태/예외/운영성
- [ ] 로딩 상태(섹션 스켈레톤, 탭 전환 스피너)가 정의되어 있다.
- [ ] 데이터 없음(첨부파일 없음, 유사 공고 없음) 상태 UI가 있다.
- [ ] 오류/재시도/안내 문구 영역이 시각적으로 고려되어 있다.
- [ ] 마감 임박(<24h) 시 강조 색상/카운트다운 변화가 명확하다.
- [ ] 원문 링크/다운로드/공유 액션의 실패 상태 표현이 있다.

### E. 상호작용
- [ ] 북마크 ON/OFF 상태가 명확히 구분된다.
- [ ] 탭 전환 시 가독성 있는 active state와 전환 효과가 있다.
- [ ] 카운트다운이 <24h일 때 실시간 갱신되는 상태를 표현한다.
- [ ] 공유 버튼 클릭 시 모달 또는 링크복사 상태가 표현된다.
- [ ] 인쇄 버튼 동작을 위한 인쇄 친화 레이아웃이 고려된다.

### F. 접근성/반응형
- [ ] heading 계층(h1/h2)이 명확하다.
- [ ] 키보드로 탭/버튼/링크 이동이 가능하도록 포커스 상태가 보인다.
- [ ] 아이콘 버튼에 ARIA 라벨을 부여할 수 있는 구조다.
- [ ] Light/Dark 모드 모두 WCAG AA 대비를 만족한다.
- [ ] Mobile에서 핵심 CTA가 첫 화면 또는 하단 고정으로 노출된다.

### G. 구현 이관 가능성
- [ ] Next.js(App Router) 컴포넌트 단위로 분해 가능한 구조다.
- [ ] 서버 데이터와 무관한 정적 UI 영역과 동적 영역이 분리되어 보인다.
- [ ] `T-023` 개발 착수 시 필요한 섹션 경계가 명확하다.
- [ ] 추후 `T-026`, `T-030`, 유료 기능 API(`T-040`) 연결 포인트가 식별 가능하다.
- [ ] 프로토타입 기준으로 FE 개발 티켓을 바로 쪼갤 수 있다.

## 승인 게이트 (Go/No-Go)
- Go 조건:
  - A~G 섹션의 체크 항목 90% 이상 PASS
  - A, B, C 섹션은 필수 항목 100% PASS
  - Mobile/Tablet/Desktop 3개 브레이크포인트 모두 확인 완료
- No-Go 조건:
  - 핵심 컴포넌트 누락
  - Flutter 계약/토큰과 충돌
  - 접근성 기준 미충족

## 진행 순서
1. `design.md` 상세페이지 프롬프트로 v1 생성
2. 본 체크리스트로 리뷰 후 FAIL 항목 기록
3. 프롬프트 보정(v2) 후 재생성
4. Go 조건 충족 시 `Task.md` 구현 태스크 착수 (`T-023` 우선)
